## 6-1: API 테스트

### 인증 플로우 테스트

#### 로그인/로그아웃
- [ ] 정상 로그인 성공
- [ ] 잘못된 비밀번호로 로그인 실패
- [ ] 존재하지 않는 계정으로 로그인 실패
- [ ] 로그아웃 후 인증 필요 API 접근 불가
- [ ] Rate Limit 동작 확인 (15분 5회)

#### 토큰 갱신
- [ ] Access Token 만료 후 Refresh 성공
- [ ] Refresh Token 만료 후 재로그인 필요
- [ ] 토큰 갱신 중 동시 요청 처리

#### 권한 검증
- [ ] 교사 전용 API에 학생 접근 → 403
- [ ] 다른 반 데이터 접근 → 403
- [ ] 다른 팀 데이터 접근 → 403

### 과제 제출 플로우 테스트

#### 개인 과제
```
1. POST /assignments (교사) → 201
2. GET /assignments/:id (학생) → 200
3. POST /submissions (학생) → 201
4. PUT /submissions/:id (학생) → 200
5. POST /submissions/:id/submit (학생) → 200
6. PUT /submissions/:id (학생) → 400 (이미 제출됨)
7. POST /submissions/:id/feedback (교사) → 200
```
- [ ] 전체 플로우 정상 동작

#### 팀 과제
```
1. POST /assignments (교사, type: 'team') → 201
2. POST /submissions (팀원 A) → 201
3. PUT /submissions/:id (팀원 B) → 200
4. POST /submissions/:id/submit (팀원 A) → 200
5. GET /submissions/:id (팀원 B) → 200 (제출 확인)
```
- [ ] 전체 플로우 정상 동작

#### 마감 처리
- [ ] 마감 전 제출 가능
- [ ] 마감 후 제출 불가 (400)
- [ ] 마감 연장 후 재제출 가능 (교사가 연장한 경우)

### 파일 업로드 테스트

#### 파일 크기 제한
- [ ] 일반 파일 20MB 이하 → 업로드 성공
- [ ] 일반 파일 20MB 초과 → 400 에러
- [ ] 동영상 100MB 이하 → 업로드 성공
- [ ] 동영상 100MB 초과 → 400 에러

#### 파일 타입 검증
- [ ] 허용된 타입 (pdf, doc, jpg 등) → 성공
- [ ] 금지된 타입 (exe, bat, sh 등) → 400 에러
- [ ] 확장자 위조 감지 (MIME 타입 검사)

#### 파일 다운로드
- [ ] 권한 있는 파일 다운로드 성공
- [ ] 권한 없는 파일 다운로드 → 403
- [ ] 존재하지 않는 파일 → 404
- [ ] Content-Disposition 헤더 확인

## 6-3: 배포 준비

### 프로덕션 빌드 테스트

#### 클라이언트 빌드
```bash
cd client
npm run build
```
- [ ] 빌드 에러 없이 완료
- [ ] `dist/` 폴더 생성
- [ ] 번들 크기 합리적 (< 500KB gzip)

#### 서버 빌드 (해당되는 경우)
```bash
cd server
npm run build  # TypeScript인 경우
```
- [ ] 빌드 에러 없이 완료

#### 프로덕션 모드 실행
```bash
NODE_ENV=production npm start
```
- [ ] HTTPS로 시작
- [ ] 정적 파일 서빙 정상
- [ ] API 정상 동작

### 환경변수 체크리스트

#### 필수 환경변수
- [ ] `NODE_ENV=production`
- [ ] `PORT=3000` (또는 원하는 포트)
- [ ] `JWT_SECRET` (강력한 랜덤 문자열)
- [ ] `JWT_REFRESH_SECRET` (다른 랜덤 문자열)

- [ ] `JWT_EXPIRES_IN` (기본: 3h)

## 주의사항

1. **데이터 백업**: 배포 전 반드시 현재 데이터 백업
2. **롤백 계획**: 문제 발생 시 이전 버전으로 복구할 수 있도록 준비
3. **점진적 배포**: 가능하면 일부 사용자에게 먼저 배포 후 전체 배포
4. **모니터링**: 배포 후 일정 시간 동안 에러 로그 모니터링
5. **사용자 공지**: 중요한 변경사항은 사용자에게 사전 공지
