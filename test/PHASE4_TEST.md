## 통합 테스트 시나리오

### 알림 플로우 테스트
```
1. 학생 A, B가 로그인하여 Socket 연결
2. 교사가 새 공지사항 작성
3. 학생 A, B 모두 실시간 알림 수신 확인
4. 학생 A가 알림 클릭 → 공지사항 페이지 이동
5. 학생 A의 알림 읽음 처리 확인
```
- [ ] 위 플로우가 정상 동작

### 공동 편집 플로우 테스트
```
1. 학생 A, B가 같은 팀 과제 문서 열기
2. 학생 A가 첫 문장 작성
3. 학생 B 화면에 실시간 반영 확인
4. 학생 B가 두 번째 문장 작성
5. 학생 A 화면에 실시간 반영 확인
6. 학생 A의 커서 위치가 학생 B 화면에 표시
7. 학생 A 연결 해제 → 커서 사라짐
8. 학생 A 재연결 → 최신 상태 로드
```
- [ ] 위 플로우가 정상 동작

### 재연결 테스트
```
1. 학생이 문서 편집 중 네트워크 끊김 시뮬레이션
2. 오프라인 상태에서 편집 계속
3. 네트워크 복구
4. 자동 재연결 및 변경사항 동기화
5. 다른 사용자의 변경사항도 반영
```
- [ ] 위 플로우가 정상 동작

## 주의사항

1. **HTTPS**: Socket.IO와 WebSocket 모두 HTTPS 환경에서 wss:// 사용
2. **프록시**: Nginx 등 리버스 프록시 사용 시 WebSocket 업그레이드 설정 필요
3. **메모리**: Yjs 문서는 메모리에 상주, 많은 동시 문서 시 메모리 관리 필요
4. **저장 타이밍**: 중요한 작업(최종 제출 전)은 명시적으로 저장 확인
